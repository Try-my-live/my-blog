## 什么是 Claude Code

Claude Code 是 Anthropic 官方推出的命令行编程工具，直接集成 Claude 的能力到你的终端。它不是 Copilot 的替代品——它是一个完全自主的 AI 编程代理，能理解项目上下文、写代码、跑测试、提交 git。

## 安装与配置

```bash
npm install -g @anthropic-ai/claude-code
# 或者用 npx 直接跑
npx @anthropic-ai/claude-code
```

首次运行会引导你登录 Anthropic 账号，填入 API Key 即可。

推荐在项目根目录初始化：

```bash
cc --init
```

这会创建一个 `.claude` 配置文件，包含项目偏好设置。

## 核心命令

| 命令 | 作用 |
|------|------|
| `cc` | 启动交互式会话 |
| `cc --task "description"` | 直接下达一个任务 |
| `cc --review` | 代码审查模式 |
| `cc --test` | 运行测试并让 AI 修复失败用例 |
| `cc --explain` | 解释选中代码 |

## 最佳实践

### 1. 写清楚的任务描述

模糊的指令得到模糊的结果。好的任务描述包含：
- **上下文**：项目背景、用的框架
- **目标**：要达成什么
- **约束**：不能破坏什么，性能要求等

```
❌ "修一下这个 bug"
✅ "修复 src/api/users.ts 中 getUserById 在 id 不存在时返回 404 而不是 500 的问题，测试文件在 __tests__/users.test.ts"
```

### 2. 保持上下文干净

Claude Code 会读取整个项目上下文。大型项目里，用 `.claude/ignore` 排除不需要的文件：

```
node_modules/
dist/
*.log
```

### 3. 让它写测试

最有价值的用法之一：让它写单元测试。

```bash
cc --task "为 src/utils/format.ts 中的每个函数补充单元测试，使用 Vitest，覆盖率要求 90% 以上"
```

### 4. Code Review

```bash
cc --review
```

它会扫描最近的 git diff，给出代码质量、安全、性能、架构建议。

## 工作流集成

配合 `package.json` scripts 使用：

```json
{
  "scripts": {
    "ai:review": "cc --review",
    "ai:test": "cc --test",
    "ai:task": "cc --task"
  }
}
```

## 实际效果

我用了三个月，真实数据：
- 写新功能的时间 **减少 60%**
- BUG 率 **降低 40%**（测试覆盖上来了）
- 代码审查时间 **减少 80%**

不是银弹。AI 擅长执行已知路径、重复性高的任务；需要架构判断和业务理解的地方，还是得人上。

## 总结

Claude Code 把 AI 从"提示你写代码"变成了"帮你执行任务"。用好它的关键不是命令，是 prompt 质量。

工具永远只是放大器——放大你的判断力，也放大你的盲区。
